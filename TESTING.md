## 0 · 前置准备

- [ ] 顶栏「当前工厂」选一家（记为 **工厂 A**）
- [ ] 进 `/admin/import`，点「下载模板」拿到 `datasheet-import-template.xlsx`
- [ ] 模板自带 3 行示例：`COB-480-24V`（灯带）/ `AL-PROFILE-10`（铝槽）/ `PWR-24V-100W`（电源），后续用例都基于它们

## 1 · M7 批量导入

### 1.1 正常路径（先预览后写库）
- [ ] 直接上传**原始模板** → Step 2 出现预览
- [ ] 摘要应为：**新增 3 / 更新 0 / 错误 0 / 配件关系 1**，规格 3 行、图片 2 行
- [ ] 产品明细表三行都是绿色「新增」chip
- [ ] 点「确认导入 3 个产品」→ toast 成功 → `/admin` 列表出现这 3 个产品

### 1.2 幂等（重复导入不翻倍）★关键
- [ ] **再次上传同一个文件** → 预览摘要变 **新增 0 / 更新 3**
- [ ] 确认导入 → `/admin` 列表产品总数**没变**（仍是 3 个，不是 6 个）

### 1.3 错误行隔离（坏行不阻断好行）
在模板副本上做这些改动，再上传看预览：
- [ ] 产品表某行**清空型号** → 该行报「缺少型号」，其余正常
- [ ] 产品表类目填 `灯泡` → 报「未知类目」
- [ ] 复制一行让**两行型号相同** → 第二行报「型号在文件内重复」
- [ ] 规格表型号填 `COB-999`（产品表没有）→ 报「型号在产品表与库中均不存在」
- [ ] 配件表「适配型号」填 `NOT-EXIST` → 报引用不存在
- [ ] 图片表 URL 填 `abc`（非 http）→ 报「图片URL 缺失或非 http(s)」
- [ ] 上述每种情况：**摘要错误行计数正确**，好行仍可确认导入
- [ ] 点「错误报告 CSV」→ 下载的文件用 Excel 打开**中文不乱码**、行号/原因对得上

### 1.4 类目中英都认
- [ ] 一行类目填 `灯带`、另一行填 `strip` → 都能通过校验（导入后两者 category 一致）

### 1.5 specs / images 不被误清 ★易错
- [ ] 先用完整模板导入 `COB-480-24V`（带 3 条规格）
- [ ] 再做一个文件：**只保留产品表**那一行 `COB-480-24V`，**规格表删空**
- [ ] 导入后打开 COB 公开页 → §02 规格**仍在**（没被空表清掉）

### 1.6 slug 冲突（跨工厂同型号）
- [ ] 顶栏切到**工厂 B**，导入同样含 `COB-480-24V` 的模板
- [ ] 工厂 A、B 两个 COB 公开页都能打开，**URL slug 不同**（第二个带后缀，不撞）

## 3 · M9 属性自动匹配后台

打开 `/admin/products/{COB-480-24V 的 id}`（从产品列表点进去）。

### 3.1 自动建议生成 ★核心
- [ ] 页面有「分类与属性」「自动匹配建议」「配件关系」三个区
- [ ] 属性区已带导入的值：类目=灯带、PCB 宽度=10mm、电压=24V、功率=14.4
- [ ] 「自动匹配建议」列出 `AL-PROFILE-10`（理由：PCB 宽度 10mm 匹配）和 `PWR-24V-100W`（理由：电压 24V 匹配）
- [ ] 已经在配件表导入过的关系**不会重复出现在建议里**

### 3.2 采纳建议 → 写入权威关系
- [ ] 点某条建议的「采纳」→ toast 成功 → 它从建议消失、出现在下方「配件关系」
- [ ] 打开 COB 公开页 → §相关产品「适配/配件」里出现刚采纳的那个

### 3.3 属性驱动建议变化
- [ ] 把 PCB 宽度改成 `8mm`，保存 → 铝槽 `AL-PROFILE-10`（10mm）建议**消失**
- [ ] 改回 `10mm` 保存 → 建议**回来**
- [ ] 把类目从「灯带」改成「电源」保存 → 建议**全部消失**（只对 strip 推荐）

### 3.4 手动关联 + 删除
- [ ] 「配件关系」区输入框填一个本厂存在的型号（输入时有候选下拉）+ 选「配件」→ 点关联 → 出现在列表
- [ ] 输入一个**不存在**的型号 → 报「当前工厂未找到型号…」
- [ ] 点某条关系的垃圾桶图标 → 从列表移除

### 3.5 越权（手动 UI 测不到，记录在案）
- [ ] 说明：切到工厂 B 后，B 的产品列表里**根本看不到** A 的产品，正常操作不可能跨厂；
      越权只能靠伪造请求 id，属代码层校验（`assertOwned`），手动覆盖不到 —— **留给集成测试或代码审查**
